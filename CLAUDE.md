# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Expo / React Native habit tracker. Three tabs (**Today**, **History**, **Challenges**), plus onboarding, a habit form, and a habit detail screen. Tracking a habit fires haptics + a synthesized chime + a scale-bounce pulse; completing a challenge shows a tier-based reward badge in a celebration modal. State lives in a Context provider persisted to `AsyncStorage`.

## SDK version — read this first

Pinned to **Expo SDK 56** (`expo ~56.0.15`, RN 0.85, React 19.2). Expo Go on the Play Store and App Store ships **SDK 54** — it will refuse to load this project. To run on device, install a matching Expo Go APK/IPA from `https://expo.dev/go` (pick SDK 56), or use `eas go` / iOS simulator.

Do NOT bump the SDK without a reason — `create-expo-app@latest` currently scaffolds SDK 57 (preview), which no store-distributed Expo Go supports.

Per `AGENTS.md`: consult the exact versioned docs at `https://docs.expo.dev/versions/v56.0.0/` before writing Expo code — the API surface has churned recently (e.g. `NativeTabs` lives at `expo-router/unstable-native-tabs`, `@expo/ui`, `expo-glass-effect`, `expo-symbols` are in the template).

## Architecture

### Router shape (expo-router file-based)

`src/app/` is the router root (`main: "expo-router/entry"`). The root `_layout.tsx` renders a `<Stack>` with these screens:

- `(tabs)/` — group, shown as one Stack.Screen. Its own `_layout.tsx` renders `AppTabs`. Contains `index.tsx` (Today), `history.tsx`, `challenges.tsx`.
- `onboarding.tsx` — full-screen, gesture-locked. Auto-shown by `OnboardingGate` (redirects `router.replace('/onboarding')` when `!onboardingComplete`).
- `habits/new.tsx` — modal-presented form. Optional `?editId=<id>` search param turns it into an edit form.
- `habits/[id].tsx` — habit detail (chart, log, edit, delete).

Typed routes are **off** (`experiments.typedRoutes: false`) — pass raw strings and cast with `as never` where needed. Turning them on requires the dev server to be running to regenerate `.expo/types/router.d.ts` before tsc will pass.

### Provider chain

Wraps everything in `src/app/_layout.tsx`, in this order:

`ThemeProvider` → `HabitsProvider` → `FeedbackProvider` → `<Stack>` + `RewardWatcher`.

- `HabitsProvider` (`src/lib/use-habits.tsx`) — single source of truth. Loads/saves the whole `Store` (habits + challenges + `onboardingComplete` + `notificationsAsked`) via `loadStore`/`saveStore`. After every store change it also calls `syncReminders(habits, challenges)` so notification schedules stay in lockstep with data.
- `FeedbackProvider` (`src/lib/feedback.tsx`) — owns a single `AudioPlayer` instance from `expo-audio` and exposes `fire('tick' | 'complete')`. `tick` = light haptic; `complete` = success haptic + chime seek-to-0 + play.
- `OnboardingGate` — thin `useEffect` in `_layout.tsx` that consults the store and redirects.
- `RewardWatcher` — thin `useEffect` in `_layout.tsx` that scans challenges for `challengeProgress(...).complete`, opens `<RewardModal>` and fires the celebration haptic + chime.

### Data model (v2)

Types live in `src/lib/habits.ts`. Key shapes:

- `Habit` — `type: 'binary' | 'count'` + `target` (binary is `1`, count is user-picked N). Completions are `Record<'YYYY-MM-DD', number>`; for binary `1` means done, for count the value is the day's count and `isDoneForDay` returns `count >= target`.
- `Challenge` — `{ habitId, days, startDate, completedAt, rewardEmoji }`. Progress = count of days-from-startDate that were `isDoneForDay`. `completedAt` acts as the "claimed" flag; the RewardWatcher won't re-fire once set.
- Storage key is `habits.v2`; `loadStore` will migrate any legacy `habits.v1` (bare array of habits with `completions: string[]`) into v2 shape as binary habits, then delete v1.

Streak/stat helpers are all in `habits.ts` — prefer `currentStreak`, `bestStreak`, `totalDoneDays`, `dayCountsForRange` over recomputing in components.

### Feedback loop (haptics + audio)

The chime WAV at `assets/sounds/chime.wav` is a synthesized bell — three sine partials (880/1318.5/1760 Hz) with an exponential decay envelope, generated by the one-off script at `../scratchpad/gen-chime.js`. If you want a different sound, rewrite the partials in that script and re-run it — the file is not from an artist and there is no "original" to preserve.

`expo-audio` requires `setAudioModeAsync({ playsInSilentMode: true })` on init to actually play on iOS silent mode — `FeedbackProvider` calls that once at mount.

### Notifications (retention hook)

`src/lib/notifications.ts` is **hard-guarded against Expo Go**:

- Top level: `import type * as NotificationsType from 'expo-notifications'` — type-only, erased at runtime.
- Runtime: `loadNotifications()` returns `null` when `Constants.executionEnvironment === ExecutionEnvironment.StoreClient` (Expo Go) and otherwise `require('expo-notifications')`. Every public API (`requestAndSetupNotifications`, `syncReminders`) short-circuits when it's null.

Why: starting SDK 53, `expo-notifications` throws at **module load** in Expo Go on Android. Any static `import` from that file would crash the whole app before rendering. Do not switch back to a static import — that regressed the app once already.

Reminders are always **local scheduled** (`SchedulableTriggerInputTypes.DAILY` per habit + one at 20:30 for active challenges), never remote push. That's why we can go without a backend.

### Charts (react-native-svg)

Three visualization components:

- `consistency-chart.tsx` — 7×5 (35-day) heatmap on habit detail. Opacity blends `backgroundSelected → text` by ratio-done.
- `big-heatmap.tsx` — 15×6 (90-day) grid on History. Same blend function, but `ratio` is habits-done/habits-total across the whole habit list for that day.
- `weekly-bars.tsx` — plain RN Views (no SVG), 7 columns, best-day highlighted.

Reward badges (`reward-badge.tsx`) are the most involved SVG: radial gradient background, dashed decorative inner ring, one perimeter dot per challenge day, day-count chip overlaid, and the habit's emoji rendered as native `<Text>` on top of the SVG (SVG `<Text>` doesn't render emoji reliably). Tiers are derived from `days` via `tierForDays()` — Bronze < 7d, Silver 7-13d, Gold ≥ 14d.

### Theming

`src/constants/theme.ts` holds `Colors` (light/dark), `Fonts`, `Spacing`, `BottomTabInset`, `MaxContentWidth`, plus a `Palette` type. Screens grab `Colors[scheme === 'unspecified' ? 'light' : scheme]` and pass that `Palette` down to helper subcomponents. **Use `Palette` (not `(typeof Colors)['light']`)** — the latter has literal-string types that only match the light palette.

`ThemedText` (fixed `type` variants — `default`/`title`/`small`/`smallBold`/`subtitle`/`link`/`linkPrimary`/`code`) and `ThemedView` (fixed `type` variants matching color keys) are the canonical building blocks. `defaultSemiBold` is **not** a valid `type`; use `style={{ fontSize: 16, fontWeight: '600' }}` when you need it.

### Platform variants

Files named `*.web.tsx` / `*.web.ts` replace the base file on web (`animated-icon`, `app-tabs`, `use-color-scheme`). `app-tabs.web.tsx` uses `expo-router/ui` `<Tabs>` — a completely different mechanism from mobile's `<NativeTabs>` — so any change to tab routes/labels has to happen in **both** files.

## Repo quirks

- `global.css` and `css.d.ts` at the repo root exist to satisfy a stray `@/global.css` import from `constants/theme.ts` and CSS-module type declarations used by `animated-icon.web.tsx`. Don't delete them without checking both.
- **Don't wrap `<Link>` around `<Pressable>`** — under `expo-router` v56 this silently drops the child on some renders (both the "+" button and the challenge banner regressed this way). Use a plain `Pressable` with `router.push(...)` in `onPress`.
- **Route casts:** because typed routes are off, `router.push('/some/route')` needs `as never` in strict-TS mode. Consistent across all screens.
- On the user's Mac, the global npm cache (`~/.npm`) has files owned by another user from a past `sudo` install. Installs have been run with `--cache <scratch dir>` to sidestep it. Fix long-term with `sudo chown -R $(whoami) ~/.npm`.
- Expo Go on Android in dev shows a floating dev-menu button in the top-right — do not place primary CTAs there or they'll be occluded. The "+ New habit" button lives at the bottom of the list for this reason.
